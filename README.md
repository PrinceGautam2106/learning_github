## Getting Started with GitHub

Here are some key concepts to help you on your journey:

### 1. Repository (Repo)
A repository is like a folder for your project. It contains all your files and tracks every change you make.

### 2. Commit
A commit is a saved snapshot of your changes. Think of it like pressing "Save" — but with a message describing what you changed.

### 3. Branch
A branch lets you work on a new feature or fix without affecting the main code. Once you're happy with your changes, you can merge them back.

### 4. Pull Request (PR)
A pull request is how you propose your changes to a project. Others can review, comment, and approve before the changes are merged.

### 5. Clone
Cloning a repository means downloading a copy of it to your local machine so you can work on it.

## Basic Git Commands

```bash
# Clone a repository
git clone <repository-url>

# Check the status of your changes
git status

# Stage changes for a commit
git add .

# Commit your changes
git commit -m "Your descriptive commit message"

# Push your changes to GitHub
git push

# Pull the latest changes from GitHub
git pull
```

## Tips for Beginners

- **Don't be afraid to make mistakes** — Git tracks everything, so you can always go back.
- **Write clear commit messages** — Future you will thank present you.
- **Use branches** — They keep your work organized and safe.
- **Read error messages** — They usually tell you exactly what went wrong.
- **Practice regularly** — The more you use Git and GitHub, the more natural it becomes.
